# fullbuild.ai — The Build Seam

## Status

Selected replacement direction. Local design prototype only. Hosting, publishing, deployment artifacts, caching, robots, Open Graph, social cards, and release verification remain excluded.

## Thesis

**The site turns possibility into matter so a recruiter remembers watching the whole product become real.**

The signature is an impossible boundary—the Build Seam—crossing the viewport. Anything passing through it changes its material law. The seam is not decoration: it controls lifecycle state, reveals content, transforms typography, and organizes future cases.

## Directions rejected

1. **Dark Proof Lab:** current near-black/vermilion/wireframe/oversized-grotesk world. Familiar creative-developer cluster; isolated WebGL object does not change navigation or proof.
2. **Proof Press / Red Thread:** warmer editorial evolution with continuous strip/thread, source marks, and a shipment seal. Coherent but retains current palette, proof vocabulary, ruled composition, and object-centric motion.
3. **Neon Metaverse / AI Control Room:** spatial WebGL world, fake telemetry, nodes, terminal language, glow, and particles. High spectacle, low specificity; interchangeable with any AI studio.

## Experience arc

### 1. Arrival — From maybe to made

- Full viewport, high-chroma flat color fields, no dominant black world.
- `fullbuild.ai`, “Independent AI product builder,” official tagline, and primary action visible immediately.
- Huge asymmetric statement: `FROM MAYBE / TO MADE.`
- Diagonal Build Seam divides two material states. Its transformation object and all four stage controls fit inside 390×844.
- Opening motion sweeps the seam through idea → design → engineering and rests. Reduced motion starts at the settled composite.

### 2. Disturbance — Cross the seam

- Pointer/drag, keyboard, or stage controls move the seam.
- Type width, artifact geometry, color field, and content labels change together.
- Visitor learns interaction through direct visual cause, not instructions.

### 3. Exploration — Worlds in queue

Three honest future-case portals prove the portfolio system before real content exists:

- `Future product / 01 — Project title pending`
- `Future research / 02 — Question pending`
- `Future experiment / 03 — Hypothesis pending`

Each portal has a distinct silhouette and motion law. None is an equal card. Replacing placeholder copy or media must not require layout redesign.

### 4. Proof — One idea, four laws of matter

Case 00 becomes a compact interactive lifecycle, not a 3,000px ledger. One artifact changes across four states:

- **Idea / fluid:** soft boundary, unstable width, buoyant fragments, cobalt + orchid.
- **Design / planar:** cut planes, crop logic, snap, milk + coral + cobalt.
- **Engineering / articulated:** routed joints, compression, visible constraints, lemon + ink + cobalt.
- **Shipped / fused:** solid depth, tactile shadow, calm surface, coral + milk + ink. Current release remains visibly gated by `production proof pending`.

The same stage IDs and truthful deployment lock remain. Visualizing the shipped law does not claim this checkout is deployed.

### 5. Payoff — The whole build

The compact Case 00 explanation shows current real decisions and source links without verification theater. The final seam attempts to close, stops at the shipment gate, and exposes exactly what is missing.

### 6. Exit — Start a world

The seam resolves into a large future contact portal: `LET’S BUILD THE WHOLE THING.` Contact value is explicitly `channel pending` until supplied. No generic three-column footer.

## Visual system

### Palette

Flat, stage-specific materials:

- Ink: `#17151F`
- Milk: `#F3F0E8`
- Cobalt: `#3347FF`
- Orchid: `#A987FF`
- Signal lemon: `#E9FF65`
- Coral: `#FF6B58`
- Mineral blue: `#8BE7FF`

Gradients may appear only inside the seam or material refraction where color change encodes transformation. No dark-neon backdrop, purple-on-white SaaS gradient, glass cards, or global grain disguise.

### Typography

- Vendor the open-source variable `Anybody` family locally (width + weight axes, OFL-1.1).
- Use width as lifecycle behavior: expansive idea, measured design, compressed engineering, balanced shipped.
- Body copy uses the same family at restrained settings; hierarchy comes from scale, width, and placement rather than a forest of uppercase labels.
- All text remains usable under fallback metrics.

### Composition

- Asymmetry, diagonal seam, edge-to-edge portals, deliberate collisions.
- No repeated eyebrow + heading + ruled-grid recipe.
- No bento, equal project cards, rounded-everything system, or decorative one-pixel-rule matrix.
- Large negative space must carry tension or transformation, never accidental emptiness.

## Motion contract

- Arrival: 1,600–2,200ms orchestrated seam sweep; interruptible.
- Stage change: 80ms anticipation, 640ms material transition, 180ms rest.
- Animate transform, clip-path, opacity, and variable-font axes deliberately; avoid `transition: all`.
- No ambient loop, cursor follower, particles, generic fade-up cascade, or idle wobble.
- Pointer affects only the directly manipulated seam/artifact.
- Reduced motion: discrete four-state foldout/composite with identical content and controls.

## Semantic and interaction contract

- Exactly one visible `h1`.
- DOM contains full story without JavaScript.
- Native links/buttons; stage controls keyboard accessible.
- `shipped` remains unavailable while `deployment.verified` is false and announces `production proof pending`.
- Placeholder cases carry `placeholder: true`, distinct visual tokens, and explicit pending language.
- No fabricated clients, employers, metrics, awards, testimonials, or outcomes.
- Internal evidence links resolve.
- Meaning survives no JS, narrow viewports, forced colors, and reduced motion.

## Browser acceptance

Required viewports: 1440×900, 1280×720, 768×1024, 390×844.

Capture arrival, mid-transition, settled lifecycle states, one interaction per placeholder portal, shipment gate, exit, keyboard focus, reduced motion, and 200% zoom. Score five-second clarity, originality, signature, composition, typography, motion, placeholder resilience, mobile equivalence, and accessibility.

Every dimension must score at least 2. Clarity, originality, signature, mobile equivalence, and accessibility must score 3. Two consecutive fresh-browser passes must find no material issue before stopping.
