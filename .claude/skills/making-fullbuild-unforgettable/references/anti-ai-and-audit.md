# Anti-AI-tell and browser-audit protocol

## AI tells are clusters

No single device proves AI authorship. Failure comes from predictable devices accumulating without a subject-specific reason.

### Structural tells

- Centered headline, supporting sentence, pill CTA, abstract visual.
- Hero → three equal cards → stats → testimonials → CTA.
- Every section uses eyebrow, giant heading, paragraph, ruled grid.
- Bento/cardification used instead of composition.
- Alternating left/right zigzag, uniform section heights, or repeated spectacle.

### Visual tells

- Inter/Space Grotesk/geometric sans chosen by habit.
- Purple gradient/glass, dark neon, cream editorial, or near-black/vermilion “creative dev” clusters.
- Wireframe orb, particle field, grid tunnel, cursor follower, fake terminal, glow, or noise used as proof of technical sophistication.
- Tiny uppercase labels, slash-separated metadata, excessive one-pixel rules, and identical radii.
- High polish with no texture, artifact, asymmetry, or content-specific irregularity.

### Behavioral tells

- Everything fades upward on scroll.
- Every element moves, but the page has no dramatic arc.
- WebGL reacts to pointer yet does not control navigation, reveal evidence, or encode state.
- Mobile deletes the signature and leaves only stacked copy.
- Reduced motion becomes a broken or empty experience.

### Language tells

- “Transform,” “elevate,” “seamless,” “innovative,” and “end-to-end” without concrete proof.
- Symmetrical three-part phrases, repeated slash labels, and defensive process narration.
- Fabricated precision: fake metrics, fake testimonials, fake client logos, fake status readouts.

## Rejection tests

- **Cluster test:** Four familiar tells in one viewport means redesign, not polish.
- **Swap test:** Replace `fullbuild.ai` with an AI agency, crypto studio, or developer portfolio. If the design still fits, reject it.
- **Decoration test:** Remove the spectacle. If meaning and navigation do not change, the spectacle is garnish.
- **Silhouette test:** Blur or squint. Distinct lifecycle states and content hierarchy must remain visible.
- **Retelling test:** A visitor must describe one unique behavior without naming the technology.
- **Placeholder test:** Replace all case content with honest placeholders. The composition must remain intentional.
- **Adjacent-evolution test:** Count inherited palette, type, silhouette, layout, metaphor, object, and motion primitives. Three retained primitives fails a reinvention brief.
- **Information-device test:** Numbers, indices, eyebrows, dividers, and coordinates must encode real information. `01–04` marks an actual four-stage sequence; a decorative index on a non-sequence is a tell.

## Browser evidence matrix

Inspect the actual checkout. Minimum widths: 1440×900, 1280×720, 768×1024, and 390×844. Capture:

- arrival before interaction;
- first transformation mid-state and settled;
- each lifecycle state;
- one placeholder case interaction;
- final/exit beat;
- keyboard focus;
- reduced motion;
- 200% zoom or equivalent narrow-content stress.

At each state score 0–3:

| Dimension | 0 | 3 |
|---|---|---|
| Five-second clarity | identity/purpose absent | identity, range, next move immediate |
| Originality | obvious category template | subject-derived and ownable |
| Signature | decoration or missing | memorable mechanism carrying meaning |
| Composition | repeated formula | controlled rhythm and spatial surprise |
| Typography | default or unstable | distinctive, legible, intentional |
| Motion | noise or garnish | dramaturgy, state, and feedback |
| Placeholder resilience | looks unfinished | feels authored and replaceable |
| Mobile equivalence | diminished concept | same idea, recomposed |
| Accessibility | meaning lost | keyboard/reduced-motion parity |

Any 0 blocks progress. Any 1 is material. Fix composition and concept before micro-detail.

## Repair loop

1. Record screenshot, viewport, state, and exact failure.
2. Name causal layer: thesis, content, composition, type, motion, rendering, or responsiveness.
3. Make the smallest change capable of removing the cause. Large conceptual failure permits replacement.
4. Repair by deepening, not accreting: before adding any element, ask whether what is already present can become more of a piece. When a state feels overworked, remove one device and re-score. New elements are the last resort, never the first move.
5. Re-run failed state and neighboring state.
6. Ask: “What still looks generated, ornamental, repeated, timid, or unexplained?”
7. Repeat.

Fresh critique agents must use a browser. Source-only review cannot validate visual quality. If the preferred browser tool is unavailable, use installed Chrome/headless Chrome and visually inspect the screenshots; disclose that substitution.

## Stopping proof

Stop only when:

- all explicit experience beats exist;
- every matrix dimension scores at least 2, with clarity, originality, signature, mobile, and accessibility at 3;
- two consecutive fresh-browser critique passes find no material issue;
- no current viewport contains a four-tell AI cluster;
- placeholder and reduced-motion modes remain complete;
- remaining proposals are alternate taste directions or new features, not clear improvements.

Passing tests alone never satisfies this gate.
